# First Aid 1.0.0 Changelog

## Overview

This release updates all maintained templates:

- `fabric1.21.1`
- `fabric1.21.11`
- `neoforge1.21.1`
- `neoforge1.21.11`

The focus of `1.0.0` is the unconscious and rescue gameplay loop, multiplayer consistency, more realistic locational damage, and server-side control over rescue behavior.

## Added

- Added a new proximity-based rescue flow for unconscious players.
- Added rescue HUD prompts and rescue progress bars for nearby valid targets.
- Added support for rescuing with `bandage` or `plaster` while sneaking beside the downed player for 8 seconds.
- Added a new execution interaction for downed players across all maintained templates.
- Added support for executing unconscious players by sneaking beside them with valid weapons or tools such as pickaxes, axes, hoes, swords, spears, tridents, or maces.
- Added `/firstaid revivewakeup on|off`.
- Added persistent server-side storage for command-controlled rescue wake-up behavior.
- Added more detailed join-time command tips for OP or otherwise authorized players.
- Added exact hit-point based projectile body-part resolution for maintained templates.
- Added directional melee hit localization so close-range attacks resolve to more believable body parts.
- Added anatomical overflow routing and overflow attenuation for body-part damage transfer.
- Added updated localization entries for English and Chinese for rescue prompts, rescue feedback, and command descriptions.
- Added dedicated bilingual HUD text and progress feedback for rescue versus execution interactions.

## Changed

- Reduced post-rescue wake-up delay to 30% of the previous duration when wake-up delay is enabled.
- Changed rescue interaction from the previous instant interaction flow to a timed rescue action.
- Changed downed-player interaction so nearby players now receive context-sensitive prompts for either rescue or execution depending on the held item.
- Changed execution handling to use a 5-second hold action before the downed player is finished.
- Changed interaction prompt presentation so rescue text and progress use green styling, while execution text and progress use red styling.
- Changed rescue completion handling to use an explicit client-to-server completion attempt as a fallback, preventing full rescue bars from stalling without resolving.
- Changed immediate rescue behavior when wake-up delay is disabled:
  - Non-critical body parts at 0 health are restored to 1 health.
  - Critical body parts at 0 health are restored to 2 health.
- Changed projectile and melee damage handling so locational hits now prefer exact body parts over broad armor-slot grouping.
- Changed overflow damage handling so lower-limb overflow travels through same-side proximal parts into the torso instead of jumping to unrelated parts.
- Changed overflow damage so each transfer step loses strength instead of propagating at full value.
- Updated command help text shown on join to describe command purpose more clearly.
- Changed body-part health display colors across all maintained templates:
  - Above 85% health now renders green.
  - Above 65% up to 85% health now renders yellow.
  - Above 40% up to 65% health now renders orange-red.
  - 40% health or below now renders red.
  - The same thresholds now apply in both the detailed health menu and the top-left player-model HUD.
- Changed visible full-health handling so treatment availability, pain feedback, and displayed health all follow the same rounded per-part presentation.
- Changed medical item usage in Creative mode so healing effects still apply without consuming the held item stack.
- Changed unconscious-player damage handling so downed players now receive 80% damage reduction across all four maintained templates.

## Fixed

- Fixed a multiplayer synchronization issue where downed players could appear standing from another player's perspective.
- Fixed unconscious damage model synchronization so tracked players receive the relevant state updates.
- Fixed rescue progress getting stuck at 8 seconds without completing in some situations.
- Fixed cases where critical body-part collapse could bypass the intended unconscious flow and kill the player immediately.
- Fixed invalid overflow paths that could redirect foot or leg overflow damage into implausible parts such as the head.
- Fixed damage distribution parity issues across `1.21.1` and `1.21.11` Fabric and NeoForge templates.
- Fixed missing hit knockback and hurt feedback across all four maintained Fabric and NeoForge templates.
- Fixed cases where a body part could look fully healed while still contributing pain and accepting another bandage or plaster application.
- Fixed healing-item application on Fabric so the active healer keeps a single-item snapshot instead of mutating the live held stack reference.

## Notes

- Rescue remains server-authoritative.
- The rescue wake-up toggle defaults to `off`.
- Maintained templates now target near-identical gameplay behavior across Fabric and NeoForge.
- All four maintained templates build successfully after these changes.
