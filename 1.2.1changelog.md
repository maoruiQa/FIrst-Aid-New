# Changelog

## 1.2.1 - 2026-04-26

Applies to:

- `fabric1.21.1`
- `fabric1.21.11`
- `fabric26.1`
- `neoforge1.21.1`
- `neoforge1.21.11`
- `neoforge26.1`
- `forge1.20.1`

### Added

- Added `/firstaid naturalregen limited2`, a natural regeneration mode based on `limited` that skips body parts at 0 health.
- Added the two-step body-part healing flow so treatment items now open a pending selection state first, then apply only after the same hand keeps right click held through the item-specific apply time.

### Fixed

- Fixed `limited` and `limited2` natural regeneration causing vanilla saturation to keep draining when there are no eligible body parts left to heal.
- Fixed `limited2` making the mod-rendered heart bar appear full after food regeneration synced vanilla health to full while zero-health legs or feet remained unrecovered.
- Fixed `fabric26.1` treatment-use countdown and treatment target feedback appearing in chat instead of the action bar.
- Fixed `fabric26.1` rescue and execution completion feedback so both actors receive action-bar messages consistently.
- Fixed body-part healing selection so releasing right click, switching items, opening other screens, or hitting other invalidation states now cancels the pending flow cleanly instead of leaving stale healing state behind.

### Changed

- Changed healing prompts so the action bar now shows the live body-part healing countdown and the centered HUD prompt explains the second hold-to-apply step.
- Changed healing items that require body-part selection to behave like a normal held-use interaction during the second step, including per-item application sound handling.
- Changed the default bandage apply time to `3000` ms.

### Notes

- All seven module builds pass with the `1.2.1` version bump.
